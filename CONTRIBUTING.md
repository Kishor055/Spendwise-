# 🤝 Contributing to SpendWise 3.0

First off, thank you for considering contributing to SpendWise! It's people like you that make SpendWise such a great tool for financial empowerment.

## 🚀 How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issue Tracker.
- Describe the bug and provide steps to reproduce.
- Include screenshots or screen recordings if possible.

### Suggesting Enhancements
- Open a "Feature Request" in the GitHub Issues.
- Explain *why* this feature would be useful to SpendWise users.

### Pull Requests
1.  Fork the repo and create your branch from `main`.
2.  Ensure your code follows our Linting rules.
3.  Include unit tests for new business logic.
4.  Update the documentation if you're changing a feature.

## 🎨 Style Guide
-   **TypeScript**: Use strict types. Avoid `any`.
-   **UI**: Follow the "Fintech Noir" design system (Dark mode, glassmorphism, high contrast).
-   **Icons**: Use `lucide-react` exclusively.
-   **Git**: Use descriptive commit messages (e.g., `feat: add net worth terminal`, `fix: resolve ocr latency`).

## 🛠️ Local Setup
1.  `npm install`
2.  Setup your `.env.local` with Firebase and Google AI keys.
3.  `npm run dev`

We look forward to your contributions!
